Clean Code

Primer

What to expect

  • You'll learn a lot about code in general
  • You'll be able to tell the difference between good code and bad code
  • You'll know how to convert bad code into good code

Why it matters?

"As long is it compiles fine and does what its supposed to do, why does it matter whether its good code or bad code?"

  • Reading : Writing :: 10:1
  • Loss in productivity
  • Regression bugs

So, why do we write bad code?

  • We're always in rush
  • We get tired/bored
  • Managers don't credit good code
  • We think working mess is much better than nothing

Who is to blame?

  • Schedules/Managers?
    • Managers do want good code implicitly
    • They consult us before committing to a deadline
  • Developers?
    • Yes, kind of :)
      • Surgeon and Patient example Its surgeon's job to sanitize the equipment before operation, even if patient forces to skip it to "speed up" the surgery.
      • Its all about being passionate
    "The only way to suceed consistently is to keep the code clean"

Clean Code is Art

  • Beautiful painting vs Just a painting
  • Delicious food vs Hostel food
  • Harmony vs Cacophony
  • Good code vs Tasteless code

"A decent programmer can look at code and tell you that its bad code. A good programmer can convert it into good code."

So, What is good code?

I like my code to be elegant and efficient. The logic should be straightforward to make it hard for bugs to hide, the dependencies minimal to ease maintenance, error handling complete according to an articulated strategy, and performance close to optimal so as not to tempt people to make the code messy with unprincipled optimizations. Clean code does one thing well.
-- Bjarne Stroustrope (Inventor of C++)
